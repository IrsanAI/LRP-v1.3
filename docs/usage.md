# Nutzungsleitfaden

## 1. Vorbereitung
- Öffne `index.html` direkt im Browser oder starte lokal einen statischen Server.

## 2. Eingabe formulieren
Beschreibe dein Ziel möglichst konkret:
- gewünschtes Ergebnis
- technische/geschäftliche Randbedingungen
- erwartete Tiefe (MVP, Architektur, Roadmap, etc.)

## 3. Analyse ausführen
Klicke auf **„Anfrage Analysieren“** und prüfe:
- Confidence-Score
- erkannte Eigenschaften
- empfohlener Weg

## 4. Optionen konfigurieren
- LLM-Ziel wählen
- Features aktivieren/deaktivieren
- Token-Budget und Sprache setzen

## 5. LRP-Protokoll generieren
Klicke auf **„LRP-Protokoll Generieren“** und nutze die Ausgabe in deinem LLM.

## 6. Iteratives Arbeiten
- Verwende die erste LLM-Antwort als Input für den nächsten Lauf.
- Konkretisiere Unschärfen über neue Constraints.
- Halte mehrere Varianten für Vergleichstests fest.

## Best Practices
- Starte mit Geschäftsziel + Nutzwert statt mit Implementierungsdetails.
- Nutze klare Abgrenzung: „Muss“, „Soll“, „Kann“.
- Plane einen zweiten Lauf für Risiko- und Failure-Mode-Analyse.
